# Language Pattern Library

在草拟每条回复时，从以下库中选择适当的句式。目标：保持全文语气一致且符合学术规范。

---

## Opening (Gratitude)

| Usage | Pattern |
|-------|---------|
| Standard | We sincerely thank the reviewer for this [insightful / constructive / valuable / crucial] [comment / suggestion / feedback / recommendation]. |
| Brief | Thanks a lot for this constructive comment. |
| With emphasis | Thank you for this crucial comment. |
| Detailed feedback | We thank the reviewer for this detailed and constructive feedback. |
| Apology needed | We sincerely apologize for the [poor quality / errors / oversight] in the original manuscript and thank the reviewer for pointing this out. |

## Describing Changes Made

| Usage | Pattern |
|-------|---------|
| Addition | According to your suggestion, we have [added / incorporated / included] [new content] into the revised manuscript. |
| Revision | We have [revised / modified / updated] [section / figure / content] to [purpose]. |
| Clarification | We have clarified [point] in [Section X] of the revised manuscript. |
| Expansion | Following your suggestion, we have significantly expanded [evaluation / analysis] to include [new scope]. |
| Improvement | We have carefully [reviewed / polished / refined] the [manuscript / section] to improve [aspect]. |

## Citing Evidence

| Usage | Pattern |
|-------|---------|
| Table reference | The results are summarized in Table X (Section Y). |
| Figure reference | As shown in Fig. X, [observation]. |
| Specific numbers | On the [dataset], our method achieves [metric] of [value], surpassing [baseline] ([value]) by [delta]. |
| Comparison | This represents a [consistent / significant / substantial] improvement over [baseline]. |
| Section reference | For the detailed [description / analysis], please refer to Section X of the revised manuscript. |

## Interpreting Results

| Usage | Pattern |
|-------|---------|
| Effectiveness | [Metric improvement] highlights the effectiveness of our [module] in [function]. |
| Robustness | These results demonstrate that our method maintains [superior / consistent] performance under [challenging conditions]. |
| Trade-off | Although our method introduces a moderate increase in [cost], the resulting performance improvements are substantial. |
| Validation | This validates that the [features / design] preserved by our method are highly effective for [downstream task]. |

## Closing / Conclusion

| Usage | Pattern |
|-------|---------|
| Suitability | Based on the above analysis, our proposed method is [suitable / well-suited] for [deployment / application] in [scenario]. |
| Confirmation | This balance of [efficiency] and [performance] confirms the suitability of our method for [scenario]. |
| Summary | In summary, [key takeaway]. |

## Cross-Referencing

| Usage | Pattern |
|-------|---------|
| Noting overlap | We note that Reviewer X also raised a similar concern regarding [topic]. |
| Pointer | For the detailed experimental results and comparative analysis, please refer to the response to Comment N. |
